Introduction

This is an example of a very simple web applications using only the tools that SBT (simple build tool) and it's echo-system offers. No need of bower, npm, or grunt. Web dependencies management is handled by the sbt-web plugin by means of webjars.

For more information, visit the project site.

How to run

Press the "run" button located at the top of the "run pane" of the activator ui, and go to http://localhost:9000 using another browser tab.

Known Bugs

If the client receives a message saying that the resource can not be found, close the activator ui and try to run this application using the activator interactive shell. But before that, remove the file <project-base>/project/sbt-ui.sbt or comment its content. Said file is added by activator during the template instantiation to activate the sbt-core-next plugin required by the activator ui. Unfortunately this plugin causes the classpath be incomplete at runtime, impeding the application to obtain the resources it needs.I have already reported said bug in the sbt-core-next plugin site.
